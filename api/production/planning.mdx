# Planning API

The Planning API manages weekly production planning, including order scheduling, personnel assignments, and deviation tracking against planned vs. actual execution.

## Authentication

- **Read operations**: `VIEW_PRODUCTION` permission
- **Planning operations**: `ASSIGN_OPERATIONS` permission
- **Deviation tracking**: `MANAGE_PRODUCTION` permission

## Get Weekly Plan

```http
GET /api/planning/week/:anio/:semana
```

**Permission:** `VIEW_PRODUCTION`

Retrieves the production plan for a specific week.

### Path Parameters

<ParamField path="anio" type="integer" required>
  Year (e.g., 2026)
</ParamField>

<ParamField path="semana" type="integer" required>
  ISO week number (1-53)
</ParamField>

### Response

<ResponseField name="id" type="integer">
  Plan ID
</ResponseField>

<ResponseField name="anio" type="integer">
  Year
</ResponseField>

<ResponseField name="semana" type="integer">
  Week number
</ResponseField>

<ResponseField name="estado" type="string">
  Plan state: Borrador, Publicado
</ResponseField>

<ResponseField name="ordenes" type="array">
  Planned production orders for the week
  <Expandable title="orden properties">
    <ResponseField name="id" type="integer">
      Assignment ID
    </ResponseField>
    <ResponseField name="orden_id" type="integer">
      Production order ID
    </ResponseField>
    <ResponseField name="proceso_id" type="integer">
      Target process ID
    </ResponseField>
    <ResponseField name="turno" type="string">
      Planned shift: T1, T2, T3
    </ResponseField>
    <ResponseField name="dia_semana" type="integer">
      Day of week (1=Monday, 7=Sunday)
    </ResponseField>
    <ResponseField name="cantidad_planificada" type="number">
      Planned production quantity
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="personal" type="array">
  Personnel assignments for the week
  <Expandable title="assignment properties">
    <ResponseField name="id" type="integer">
      Assignment ID
    </ResponseField>
    <ResponseField name="persona_id" type="integer">
      Person ID
    </ResponseField>
    <ResponseField name="proceso_id" type="integer">
      Assigned process ID
    </ResponseField>
    <ResponseField name="turno" type="string">
      Assigned shift
    </ResponseField>
    <ResponseField name="dia_semana" type="integer">
      Day of week
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash cURL
curl -X GET https://api.example.com/api/planning/week/2026/10 \
  -H "Authorization: Bearer <token>"
```

## Create Plan

```http
POST /api/planning/create
```

**Permission:** `ASSIGN_OPERATIONS`

Creates a new weekly production plan.

### Request Body

<ParamField body="anio" type="integer" required>
  Year
</ParamField>

<ParamField body="semana" type="integer" required>
  ISO week number (1-53)
</ParamField>

### Response

Returns the newly created plan object with status `Borrador`.

### Example Request

```bash cURL
curl -X POST https://api.example.com/api/planning/create \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "anio": 2026,
    "semana": 11
  }'
```

## Publish Plan

```http
POST /api/planning/publish
```

**Permission:** `ASSIGN_OPERATIONS`

Publishes a plan, making it visible to production operators and locking further modifications.

### Request Body

<ParamField body="id" type="integer" required>
  Plan ID to publish
</ParamField>

### Business Rules

- Once published, a plan cannot be unpublished (create a new plan instead)
- Published plans are used by the Bitácora system for shift inheritance
- All personnel and order assignments should be finalized before publishing

### Example Request

```bash cURL
curl -X POST https://api.example.com/api/planning/publish \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"id": 5}'
```

## Record Deviation

```http
POST /api/planning/deviation
```

**Permission:** `MANAGE_PRODUCTION`

Records a deviation from the planned schedule.

### Request Body

<ParamField body="plan_id" type="integer" required>
  Plan ID
</ParamField>

<ParamField body="bitacora_id" type="integer" required>
  Bitácora ID where deviation occurred
</ParamField>

<ParamField body="orden_id" type="integer" required>
  Order ID affected
</ParamField>

<ParamField body="proceso_id" type="integer" required>
  Process ID
</ParamField>

<ParamField body="motivo_id" type="integer" required>
  Deviation reason ID (from motivos catalog)
</ParamField>

<ParamField body="cantidad_planificada" type="number" required>
  Originally planned quantity
</ParamField>

<ParamField body="cantidad_real" type="number" required>
  Actual produced quantity
</ParamField>

<ParamField body="comentarios" type="string">
  Additional comments or context
</ParamField>

### Example Request

```bash cURL
curl -X POST https://api.example.com/api/planning/deviation \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "plan_id": 5,
    "bitacora_id": 43,
    "orden_id": 12345,
    "proceso_id": 1,
    "motivo_id": 2,
    "cantidad_planificada": 1500,
    "cantidad_real": 1200,
    "comentarios": "Paro no programado por falla eléctrica"
  }'
```

## Get Plan KPIs

```http
GET /api/planning/kpi/:plan_id
```

**Permission:** `VIEW_PRODUCTION`

Calculates performance metrics for a completed week.

### Path Parameters

<ParamField path="plan_id" type="integer" required>
  Plan ID
</ParamField>

### Response

<ResponseField name="cumplimiento_global" type="number">
  Overall plan fulfillment percentage (0-100)
</ResponseField>

<ResponseField name="total_planificado" type="number">
  Total planned production across all orders
</ResponseField>

<ResponseField name="total_producido" type="number">
  Total actual production
</ResponseField>

<ResponseField name="ordenes_completadas" type="integer">
  Number of orders meeting 100% target
</ResponseField>

<ResponseField name="ordenes_totales" type="integer">
  Total orders in plan
</ResponseField>

<ResponseField name="desviaciones_count" type="integer">
  Number of deviations recorded
</ResponseField>

<ResponseField name="por_proceso" type="array">
  Fulfillment breakdown by process
</ResponseField>

### Example Response

## Planning Workflow

<Steps>
  <Step title="Create Plan">
    Create a plan for next week (e.g., Week 11)
  </Step>
  <Step title="Assign Orders">
    Assign production orders to specific days, shifts, and processes
  </Step>
  <Step title="Assign Personnel">
    Assign available personnel to cover all shifts and processes
  </Step>
  <Step title="Review & Adjust">
    Review capacity, balance workload, adjust as needed
  </Step>
  <Step title="Publish">
    Publish the plan to make it active and visible to operators
  </Step>
  <Step title="Execute">
    During the week, operators see planned orders in their bitácoras
  </Step>
  <Step title="Track Deviations">
    Record deviations when actual differs from plan
  </Step>
  <Step title="Analyze KPIs">
    After week completion, review KPIs to improve future planning
  </Step>
</Steps>

## Best Practices

<Tip>
**Advance Planning**: Create plans at least 3-5 days before the target week to allow for material procurement and personnel coordination.
</Tip>

<Check>
**Capacity Validation**: Ensure planned quantities are realistic given:
- Machine capacity and availability
- Personnel skill levels and availability
- Material lead times
- Maintenance schedules
</Check>

<Warning>
**Deviation Tracking**: Always record deviations with specific reasons. This data drives continuous improvement and helps identify systemic bottlenecks.
</Warning>
